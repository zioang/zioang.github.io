Java虚拟机（一）

2020-07-13
JVM / Java

说明

  本系列笔记主要参考了 “深入理解Java虚拟机 - JVM高级特性与最佳实践”、“实战Java虚拟机 - JVM故障诊断与性能优化(第二版)” 和相关博客。因此，若无特别说明jdk的版本，那么所使用的 jdk 版本为 1.7。

Java堆

  Java堆是被所有线程所共享的一块区域( Java虚拟机所管理的内存中最大的一块 )，几乎所有的对象实例都存放在堆中( 由于栈上分配、标量替换优化技术的出现，导致没那么的绝对 )，且Java堆是完全 自动化管理 的，通过垃圾回收机制，垃圾对象会被自动清理，而不需要人为释放。

  根据垃圾回收机制的不同，Java堆有可能会拥有不同的结构。其中，最为常见的一种是将其分为 新生代和老年代。其中，新生代存放新生对象或年龄不大的对象，老年代则存放老年对象。如下表所示，新生代可分为 Eden、S0、S1，其中 S0 和 S1 被称为幸存者区 ( Survivor Space )， 它们分别又被称为 From 和 To 区域，它们是两块大小相等、可以互换的内存空间。
